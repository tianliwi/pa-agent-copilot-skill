【文件定位】阶段一三层市场诊断总框架：先定位周期状态，再评估市场背景，再做逐K信号确认，最终输出JSON供阶段二策略路由。

【总原则】
- 三层顺序固定：周期频谱定位 → 市场背景评估 → 逐K信号确认；三层互证后才能进入阶段二。
- 阶段一只做诊断、概率评估、风险提示与路由信息，不直接给完整下单/止损/止盈方案。
- 先判断市场位置，再选策略，再执行决策；不预测方向，只评估概率分布；核心依据是K线结构而非指标。
- 周期是连续频谱而非二元分类；状态转换渐进而非突变；同一市场在不同时间框架可处于不同状态。
- 必须同时看HTF/MTF/LTF，并在每根K线收盘后持续更新 cycle_position、direction、strategy_files_needed 等判断。

【第一层：周期频谱定位】
- 任务：判断当前市场处于8种周期频谱中的哪一类，并给出方向偏好、概率、常见陷阱与阶段二可能需要的参考文件。
- 思维要求：频谱思维、量化思维、嵌套思维、动态思维、诊断思维。

【八种周期状态】

【状态1：尖峰 spike】
- 定义：最强趋势形态；连续同方向强K线至少5根，回撤极小，通常仅1~3根K线内，斜率陡。
- 识别：
  - 连续5根以上趋势棒，实体大、影线短、几乎无回撤。
  - 20EMA快速上扬/下压并与价格明显拉开。
  - 成交量较前期放大。
  - 波段间几乎无重叠，情绪呈单向贪婪/恐惧。
- 等式：尖峰 = 突破 = 缺口 = 高潮。
- 后续概率：60%恢复为通道；30%进入交易区间；10%直接反转。
- 规则：
  - 尖峰中反转尝试胜率极低；除非出现“趋势线突破 + 极点测试失败”双重证据，否则只做顺势。
  - 连续高潮（2~3段尖峰无回撤）后，通常出现两条腿调整；这是正常趋势整理，不等于反转。

【状态2：微型通道 micro_channel】
- 定义：2~10根K线构成、几乎无回撤的单向移动；斜率极大，回撤极小或没有。
- 识别：
  - 2~10根连续同向K线，很少出现明显影线。
  - 20EMA紧贴价格，或价格轻微穿越均线后立即收回。
  - 波段高低点距离极小。
  - 常是更小时间框架上的尖峰表现。
- 关键概率：第一个突破尝试失败率 > 80%。
- 交易含义：
  - 只做顺势，不逆势。
  - 第一个突破失败后，顺势入场胜率更高。
  - 不在微型通道内做反转。

【状态3：紧凑通道 tight_channel】
- 定义：斜率大、回撤小且短暂的通道；回撤常仅2~4根K线。
- 识别：
  - 通道结构清晰，波段等距且方向一致。
  - 20EMA与价格保持紧密联系，回踩EMA即获支撑/阻力。
  - 每次回撤都浅且恢复快。
  - 上下轨清晰可见。
- 交易含义：
  - 只做顺势，不做逆势。
  - 回踩均线是顺势机会。
  - 边界突破需谨慎，通道突破后常出现反向运动。
  - 上涨通道在HTF上是空头旗形；下跌通道在HTF上是多头旗形。

【状态4：常规通道 normal_channel】
- 定义：斜率适中、回撤适中的标准通道；波段结构清晰，波段利润空间可观。
- 识别：
  - 每个波段约10~30根K线，方向清晰。
  - EMA斜率适中，与价格保持适度距离。
  - 回撤深度通常30%~50%。
  - 上下轨可由趋势线画出。
- 交易含义：
  - 主要顺势；可少量逆势刮头皮。
  - 逆势前提：价格到达通道边界且出现明确反转信号。
  - 上涨通道=HTF空头旗形；下跌通道=HTF多头旗形。
  - 通道内波段交易通常比纯趋势跟踪风险更低。

【状态5：宽通道/台阶 broad_channel / staircase】
- 定义：斜率小、波段大、回撤深，但整体方向仍在。
- 识别：
  - 每个波段30~50根K线或更多，波段幅度大。
  - EMA斜率小，趋近水平。
  - 回撤深度通常50%~78.6%。
  - 上下轨宽，价格在其中大幅振荡。
- 关键特征：几乎每次突破后都有突破测试（breakout test）。
- 交易含义：
  - 可以双向交易，每个波段都可独立处理。
  - 边界突破后不要追，等待突破测试/回踩再顺势入场。
  - 台阶型是宽通道特例：高点更高、低点更高（或相反）的逐步推进结构。

【状态6：趋势型交易区间 trending_tr】
- 定义：趋势与区间之间的过渡态；有方向偏好但每个波段都可能反转，推进断续。
- 识别：
  - 波段重叠明显，没有流畅趋势。
  - EMA较平。
  - 边界不清晰或边界随时间移动。
  - 可能仅有微弱方向偏好（低点渐高/高点渐低）。
- 交易含义：
  - 这是最危险区域之一。
  - 方向偏好不可靠，每个波段都可能反转。
  - 大幅减少交易频率，或等待趋势重新明确。
  - 若交易，优先止损单，不轻易用限价单。

【状态7：交易区间 trading_range】
- 定义：典型双向震荡；上下边界清晰，价格在边界间往返；核心特征是不确定性。
- 识别：
  - 波段大量重叠，无清晰方向。
  - EMA近乎水平，与价格频繁交叉。
  - 上下边界由波段极点清晰形成。
  - 常见边界附近放量、区间内部缩量。
- 关键统计：约80%的突破尝试失败。
- 交易含义：
  - 核心策略是低买高卖的刮头皮/潮退交易。
  - 区间内追突破是最常见亏损模式。

【状态8：极端交易区间 extreme_tr】
- 定义：最弱市场状态；方向感完全丧失，均线近乎水平，价格在极窄范围内大量耗时振荡。
- 识别：
  - K线高度重叠，影线频繁，价格移动缓慢。
  - 20EMA几乎水平，与价格纠缠。
  - 时间消耗明显但空间扩展很小。
  - 常见于重大新闻/数据前的不确定阶段。
- 判定原则：
  - 不设硬量化门槛，依赖整体定性强度综合判断。
  - 当市场极度混乱、方向感完全丧失、任何入场期望值接近零或为负时输出 extreme_tr。
  - 宁可略晚确认，也不要过早输出而错过本可交易的市场。
- 交易含义：最好不交易。

【第一层通用转换规则】
- 转换期将交易频率降至正常的一半。
- 转换确认后再调整策略，不提前猜测转换方向。
- 若转换信号不明，维持当前策略直到信号清晰。
- 每次转换后必须重新评估HTF/MTF/LTF的周期位置。

【第一层检查清单】
- 方向性：
  - 20EMA方向（上/下/平）。
  - 最近20根K线整体方向。
  - 波段高点是否抬高、低点是否降低。
  - K线重叠度（重叠多偏区间，重叠少偏趋势）。
- 波动性：
  - 最近10个波段的平均幅度。
  - 回撤平均深度（相对波段幅度）。
  - 通道宽度（窄/适中/宽）。
  - 是否存在尖峰（连续5根以上强趋势棒无回撤）。
- 惯性：
  - 当前惯性方向（上/下/无方向）。
  - 是否有初步反转证据（趋势线被触及但未突破？极点接近但未失败？）。
  - HTF趋势方向。
- 嵌套框架：
  - HTF周期位置。
  - 交易时间框架周期位置。
  - 入场时间框架机会。
  - 大框架方向偏好是否与小框架入场方向一致。
- 输出要求：
  - 综合得到 cycle_position，并记录 diagnosis_confidence（0~100整数）。
  - strategy_files_needed 仅作阶段二参考，实际加载由程序按 cycle_position、direction、spike_stage、alternative_cycle_position、market_phase、detected_patterns 静态路由决定。

【第一层多时间框架嵌套】
- 核心原则：所有趋势包含小区间，所有区间包含小趋势。
- 三层框架：
  - HTF（日线/4小时）：判断整体方向和周期位置，识别趋势线/通道/区间。
  - MTF（常用60分钟）：在HTF背景下识别当前波段与具体方向。
  - LTF（常用5/15分钟）：等待结构信号，精确入场时机与止损位置。
- 常见场景：
  - HTF尖峰 + LTF回撤：整体极强，回撤是顺势机会。
  - HTF交易区间 + MTF微型通道：区间中的小趋势，常为刮头皮机会。
  - HTF宽通道 + LTF尖峰突破：可能是趋势加速，等待回踩测试后顺势。
  - HTF趋势 + MTF趋势型交易区间：多为趋势内整理，等区间结束后顺势。
- 核心规则：
  - 大框架方向偏好与小框架入场方向一致才交易。
  - 小框架若逆HTF，必须有更强理由（区间边界/楔形反转等）。
  - HTF若为交易区间，MTF方向偏好不可靠，每个波段独立看待。
  - HTF若为尖峰，MTF回撤通常是顺势机会，不是反转。
- 分析流程：
  - 先确定HTF/MTF/LTF组合。
  - HTF观察最近50~100根K线，识别8状态之一、关键趋势线、均线与方向偏好。
  - MTF观察最近30~50根K线，识别当前周期状态及其与HTF的关系。
  - 验证HTF与MTF方向一致性；一致则高置信度，不一致则降级处理。
  - LTF只确认是否有值得阶段二评估的结构信号，不在此层输出具体下单。
  - 每根K线收盘后持续更新；若周期变化，更新 cycle_position、direction、strategy_files_needed，必要时令 gate_result=wait。

【第一层例外与陷阱】
- 陷阱：
  - 把常规通道当强趋势；上涨通道在HTF上是空头旗形。
  - 在趋势型交易区间做趋势跟踪。
  - 忽略HTF方向，只看LTF。
  - 尖峰后立刻做反转；尖峰后最常见是通道整理（60%），不是反转（10%）。
  - 周期转换期仍沿用旧策略。
  - 在宽通道中追突破；正确做法是等突破测试。
- 例外：
  - 交易区间中的尖峰级突破（连续3~5根强趋势棒）成功率更高。
  - “趋势线突破 + 极点测试失败”双重确认可显著提高反转概率。
  - 紧凑通道突破若伴随放量且与HTF方向一致，可用小仓位尝试。
- 输出约定：第一层结论只输出JSON字段，不输出文本报告。

【第二层：市场背景评估】
- 任务：在周期定位基础上评估HTF背景，判断方向偏好、强弱、阶段、位置与嵌套一致性，决定如何解读信号可靠性。
- 核心原则：Context is everything。先判断背景，再解读信号；背景是概率分布，不是确定答案；背景随每根K线收盘持续更新。
- 区别：
  - 市场周期：市场结构类型（尖峰/通道/区间等）。
  - 市场背景：该结构下信号应如何解读、可靠性如何、顺逆势偏好如何。

【背景五维】

【维度1：趋势方向】
- 判断方法：
  - EMA上倾/下倾/水平。
  - 波段高低点结构：高低点同步抬高=上升；同步降低=下降；交错=无趋势。
  - 最近20根K线收盘重心上移/下移/横移。
  - 价格持续在EMA上方=多头背景；持续在下方=空头背景；反复穿越=中性。
- 置信度分级：
  - 强多头：EMA明显上倾 + 价格远离EMA + 波段结构清晰 + 重心上移。
  - 弱多头：EMA微上倾 + 价格贴EMA + 结构模糊。
  - 中性：EMA水平 + 价格围绕EMA震荡。
  - 弱空头：EMA微下倾 + 价格贴EMA + 结构模糊。
  - 强空头：EMA明显下倾 + 价格远离EMA + 波段结构清晰 + 重心下移。

【维度2：趋势强度】
- 判断方法：
  - 重叠少=强趋势，重叠多=弱趋势。
  - 回撤小（1~3根）=强趋势；回撤大（5根以上）=弱趋势。
  - 同方向趋势K线比例越高，趋势越强。
  - 价格远离EMA=强，贴近EMA=弱；2HM（2小时以上远离EMA）=超强趋势。
  - 波段幅度大且方向一致=强；幅度小且方向杂乱=弱。
- 关键规则：
  - 最长的趋势K线若是逆趋势K线，反而常说明原趋势很强（逆势形态通常“看起来更好”，更易诱空/诱多）。
  - 未出现连续两根K线收盘于EMA另一侧前，不轻言趋势结束。
  - 大幅跳空后若不迅速反转，常是当日强趋势起点。

【维度3：趋势阶段】
- 初期（尖峰阶段）：
  - 刚突破前趋势/凝滞区趋势线。
  - 连续强趋势K线，回撤极小。
  - 动能最强、顺势概率最高。
  - EMA快速转向并与价格拉开。
  - 第一次回撤常是风险收益比最优的顺势机会。
- 中期（通道阶段）：
  - 尖峰后进入震荡整理并恢复趋势。
  - 形成可辨识通道。
  - 回撤更大更久，但方向仍明确。
  - EMA保留斜率，回踩常获支撑/阻力。
  - 上涨通道在HTF上是空头旗形；下跌通道在HTF上是多头旗形。
- 末期（高潮/衰竭）：
  - 趋势轨道线被穿越。
  - 出现过长K线或高潮式推进。
  - 反转K线开始出现但仍可能失败。
  - EMA斜率趋平。
  - 多数趋势以趋势轨道线突破结束；连续高潮后常有两条腿调整。

【维度4：位置评估】
- 区间中位置：
  - 上部（上1/3）：做空概率高于做多。
  - 中部（中1/3）：方向最不清晰，少做或只刮头皮。
  - 下部（下1/3）：做多概率高于做空。
  - 边界附近：支撑/阻力最明确，是潮退交易最佳区域。
- 趋势中位置：
  - 起点附近：顺势概率最高，逆势风险最大。
  - 中段：顺势概率仍高，回调常为入场机会。
  - 末端/接近趋势轨道线：逆势概率开始上升，但仍需确认。
- 关键位置：
  - EMA：上方偏多、下方偏空、附近不明。
  - 趋势线：测试中是关键时刻，突破与否决定后续方向。
  - 前高/前低：接近前高=阻力；接近前低=支撑；突破前高/低=可能新趋势。
  - 缺口：未回补时保留支撑/阻力；已回补则该作用消失。

【维度5：时间框架嵌套】
- HTF决定大背景；HTF若为上升趋势，则MTF/LTF做空信号可靠性降低；反之亦然。
- MTF负责识别当前波段；其回调在HTF趋势背景下通常是顺势机会。
- LTF负责精确入场；LTF信号须与HTF方向偏好一致才优先考虑。
- 核心规则：
  - 大框架方向偏好必须与小框架入场方向一致。
  - HTF强趋势下，MTF/LTF回撤通常不是反转信号。
  - HTF若为交易区间，MTF方向偏好不可靠。
  - 上涨通道即使仍涨，在HTF上依然意味着上方阻力存在。

【趋势背景判断】
- 趋势初期：
  - 特征：趋势线刚突破、第一次回撤、连续强趋势棒、跳空不快反转、2HM。
  - 含义：顺势概率最高，逆势胜率极低，反转尝试多失败。
  - 决策：只做顺势；每次回撤都可视为顺势机会；仓位可偏大；错过第一次回撤则等第二次（第二高/低）。
- 趋势中期：
  - 特征：通道形成；回撤30%~50%（常规）至50%~78.6%（宽通道）；EMA仍有斜率；波段有重叠但方向清晰。
  - 含义：顺势概率仍较高；回调更常是顺势机会而非反转。
  - 决策：主要顺势；常规通道可少量逆势刮头皮；宽通道可双向波段；边界突破后等回踩再入；仓位适中。
- 趋势末期：
  - 特征：趋势轨道线穿越、高潮走势、反转K线开始出现、EMA变平、回撤加深。
  - 含义：顺势概率下降，逆势概率上升但仍需确认；高潮后第一次回撤既可能恢复趋势也可能开启反转。
  - 决策：缩小顺势仓位，不再加仓；关注反转但不抢跑；等待“趋势线突破 + 前极点测试失败”；连续高潮后预期两条腿调整。

【区间背景判断】
- 区间上部：
  - 特征：到达上边界，出现上影线/阴线反转，多头获利了结、空头入场。
  - 含义：做空概率高于做多；向上突破多失败；反转K线/上影线利于潮退交易。
  - 决策：在上部找空；不追突破；区间顶部短K线常是潮退机会。
- 区间下部：
  - 特征：到达下边界，出现下影线/阳线反转，空头获利了结、多头入场。
  - 含义：做多概率高于做空；向下突破多失败。
  - 决策：在下部找多；不追空；区间底部短K线常是潮退机会。
- 区间中部：
  - 特征：价格处于中间区域，EMA水平，反复穿越，重叠多、影线频繁。
  - 含义：方向最不确定，信号可靠性差，常见铁丝网。
  - 决策：减少交易或暂停；若交易仅小仓位刮头皮；等待价格到边界。

【关键位置背景评估】
- EMA位置：
  - 价格在EMA上方且EMA上倾：多头背景；每次回踩EMA是顺势买点；第一根收在EMA下方不一定反转；需连续两根收于另一侧才考虑趋势变化。
  - 价格在EMA下方且EMA下倾：空头背景；每次反弹至EMA是顺势卖点；2HM后第一次触碰EMA几乎总会产生趋势中结构并导致新极值。
  - 价格在EMA附近且EMA水平：中性背景，等突破确认。
  - EMA缺口K线：高点低于EMA（多头背景中的弱信号）或低点高于EMA（空头背景中的弱信号）；常是回撤序列的重要节点。
- 趋势线位置：
  - 测试趋势线时，不预判逆势；趋势线被突破也不等于反转，通常先恢复趋势并测试前极值。
  - 只有趋势线突破后再出现前极点测试失败，才是可靠反转。
  - 趋势线假突破常制造趋势中顺势入场机会，是逆势交易者被套的位置。
- 前高/前低：
  - 突破后1~2根K线内回撤成失败，不一定是反转，也可能只是突破回踩。
  - 突破回踩（杯子与把手）是高质量顺势结构。
  - 在趋势背景下，对前高/前低的测试失败也可能只是正常延续；在区间背景下则常是潮退机会。
- 缺口位置：
  - 未回补：沿缺口方向仍有支撑/阻力；跳空缺口可作旗杆，后续易形成双底/双顶旗形回踩结构。
  - 已回补：支撑/阻力消失，常是趋势减弱与市场不确定上升的信号。

【突破背景判断】
- 真突破背景：
  - 发生于已确立强趋势中。
  - 突破K线强（长实体、短影线、收盘强）。
  - 放量。
  - 突破后数根K线维持方向。
  - 此前有趋势线突破作为前奏。
- 假突破背景：
  - 发生于交易区间或趋势型交易区间。
  - 突破K线弱（短实体、长影线、收盘弱）。
  - 1~2根K线内回到突破点内侧。
  - 无前序趋势支撑。
  - 突破发生在EMA错误一侧。
- 区间突破可靠性：
  - 约80%区间突破失败。
  - 区间持续越久，突破前的假突破越多。
  - 连续3~5根强趋势K线构成的尖峰级突破是例外，成功率更高。
  - 追突破不如等回踩测试安全。
  - 若突破失败后再次失败（失败的失败）= 突破回踩，是高质量顺势机会。
- 趋势线突破确认：
  - 必要条件：收盘在趋势线另一侧。
  - 通常还需：前极点测试失败（无法创新高/新低）。
  - 只有条件一时，不急于反转；趋势常恢复并测试前极值。
  - 两条件同时满足时，反转概率显著提高。
  - 少数特例：趋势线突破动能极强，几乎不测试前极值；但这不是常态。
  - 强趋势中多数趋势轨道线穿越与反转都会失败。

【背景如何决定信号可靠性】
- 同样的反转K线：
  - 趋势末端：可靠性高，可考虑逆势。
  - 趋势中期：可靠性中等，可能只是正常回撤。
  - 区间中部：可靠性极低，多为随机波动。
  - 尖峰中：可靠性最低，不做反转。
- 同样的趋势K线：
  - 尖峰中：强确认，顺势概率极高。
  - 通道中：趋势延续概率较高，回调中的趋势K线是顺势机会。
  - 区间中：可能只是区间波动，不代表新趋势开始。
- 背景与仓位：
  - 强背景（尖峰初期 + 顺势 + HTF一致）：仓位可偏大，止损可略宽，尽量保留大部位。
  - 中背景（通道中期 + 顺势 + HTF一致）：仓位适中，可部分用于剥头皮。
  - 弱背景（区间 + 方向不明 + HTF不一致）：缩小仓位或不交易，止损更紧，仅适合刮头皮。

【第二层分析流程】
- 1 判断趋势方向：EMA方向、波段结构、收盘重心，得出多/空/中性及强中弱。
- 2 判断趋势强度：重叠度、回撤幅度、2HM、波段质量。
- 3 判断趋势阶段：初期/中期/末期，关注趋势线突破、通道、轨道线穿越、EMA变平。
- 4 判断价格位置：区间上中下、趋势起中末、EMA/趋势线/前高低/缺口关系。
- 5 判断时间框架嵌套：HTF周期位置与方向偏好、MTF波段、LTF方向是否一致。
- 6 综合五维，输出背景类型与背景置信度。
- 7 依据背景决定策略方向、仓位级别、入场方式、止损框架与需避免行为。
- 8 每根K线收盘后重评五维；若趋势线突破、均线方向改变、波段结构变化，则立即更新背景。

【第二层例外与陷阱】
- 陷阱：
  - 把上涨通道误判为强多头背景。
  - 在交易区间里做趋势跟踪。
  - 忽略HTF方向，被LTF小趋势迷惑。
  - 在强趋势中过早反转。
  - 把趋势中期回撤当成反转。
  - 在区间中部过度交易。
  - 追突破。
- 例外：
  - 尖峰级区间突破可考虑追入。
  - “趋势线突破 + 极点测试失败”是高可靠反转背景。
  - 趋势末期的反转K线可靠性显著高于趋势中期。
  - 趋势线假突破可形成高质量顺势机会。

【第三层：逐K信号确认】
- 任务：用信号棒/入场棒/确认棒验证前两层诊断，并在K线收盘后寻找可交给阶段二执行的具体结构条件。
- 核心立场：逐根K线分析；每根收盘后重新评估；不预测，只依据当前结构评估后续最可能路径与概率。
- 核心纪律：
  - 每根K线都提供信息。
  - 每根K线要么更接近趋势K线，要么更接近十字星/凝滞区K线。
  - 信号线质量决定交易可靠性。
  - 不在收盘前进场。
  - 二次进场通常优于第一次。
  - 趋势K线必须有跟随；无跟随则可能失败。
  - 所有判断以收盘价为核心。

【K线分类：趋势K线 vs 十字星】
- 趋势K线：
  - 有明显实体。
  - 实体长度等于或长于过去5~10根K线中值。
  - 阳线常接近开=低、收=高；阴线常接近开=高、收=低。
  - 影线更短，显示一方主导。
  - 阳线收盘应与前几根收盘/高点一致或更高。
- 十字星/凝滞区K线：
  - 实体很短或没有。
  - 多空均衡。
  - 尺度依时间框架而变，但结构含义相同。
  - 单根十字星表示无人控制；连续十字星若收盘/高低点持续递增或递减，也可构成趋势。
- 关键区分逻辑：
  - 目的只是判断哪方占优。
  - 实体短的模糊K线要结合环境判断。
  - 趋势K线不必然意味着市场在趋势中；无跟随的强阳/强阴可能是假突破。
  - 十字星也不一定表示无趋势；十字星序列可形成趋势。
- 决策对应：
  - 趋势K线：顺势信号质量更高，仓位可略放大。
  - 十字星：更谨慎，需更多确认；凝滞区中的十字星通常不是好信号。
  - 强趋势中的趋势中结构里，十字星也可成为信号线。
  - 凝滞区高点下方卖出十字星、低点上方买入十字星，本质类似做大凝滞区边界交易。

【信号线 Signal Bar】
- 定义：促使交易者准备进场的结构K线；只有事后才被确认。
- 识别规则：
  - 一根K线收盘且相关交易结构成立后，前一根才可视为信号线。
  - 几乎每根K线都可能成为潜在信号线，但多数不会真正成为有效信号线。
  - 新手只在“信号线同时也是趋势K线”时交易更安全。
  - 做多优先选阳性趋势K线作信号线；做空优先选阴性趋势K线作信号线。
- 进场规则：
  - 在信号线极点外1个报价单位（1 tick）设置止损入场单。
  - 股票可在潜在信号线外几个报价单位设置止损单。
  - 市场有时只越过1个报价单位就反转，注意1 tick陷阱。
- 与方向关系：
  - 信号线若与交易方向同向且处于清晰趋势背景，胜率显著提升。
  - 反方向趋势K线也能当信号线，但风险更大。
  - 若信号线是反方向十字星/趋势K线，亏损概率上升。
  - 等到目标方向至少已控制信号线后再进场，更优。
- 可靠性排序：
  - 1 同方向趋势K线。
  - 2 标准反转K线。
  - 3 短K线/内包线（需环境支持）。
  - 4 反方向十字星或趋势K线。

【进场K线 Entry Bar】
- 定义：触发信号线外止损单的当前K线。
- 规则：
  - 进场K线收盘后立刻评估交易状态。
  - 收盘强势且与持仓方向一致 → 保留更多仓位。
  - 收盘弱势或呈十字星/反向 → 收紧止损或减仓。
  - 若进场K线在收盘前最后几秒突然加速，可减少剥头皮，更多考虑长线空间。
- 止损管理：
  - 通常设在进场K线极点外1个报价单位。
  - 进场K线收盘后可考虑收紧止损。
  - 若收盘极强，可把止损移至盈亏平衡附近。

【反转K线 Reversal Bar】
- 最佳熊牛反转K线（看多）：
  - 开盘在前一根收盘附近或更低。
  - 收盘高于开盘且高于前一根收盘。
  - 下影线约占全长1/3~1/2。
  - 上影线很短或几乎没有。
  - 与前1~数根K线重叠小。
- 最佳牛熊反转K线（看空）：
  - 开盘在前一根收盘附近或更高。
  - 收盘低于开盘且低于前一根收盘。
  - 上影线约占全长1/3~1/2。
  - 下影线很短或几乎没有。
  - 与前1~数根K线重叠小。
- 强度判断：
  - 若信号线本身也是与你交易方向同向的趋势K线，则更强。
  - 做多反转K线若收盘远高于开盘，胜率更高；做空反之。
  - 反转K线若明显短于前几根，尤其实体很短，逆势能量不足，风险大。
  - 实体很长且其他条件吻合时，风险显著降低。
- 陷阱识别：
  - 与前几根大幅重叠或影线极长 → 更可能只是凝滞区一部分。
  - 实体很短但整根很长（大十字星）→ 一根K线级别的凝滞区。
  - 顶部牛熊反转K线若下影过长、底部熊牛反转K线若上影过长 → 逆势一方不够坚定。
  - 强趋势中常见收盘前几秒由看似强反转变成失败反转。
- 使用条件：
  - 强趋势里做逆势，必须等趋势线被打破且出现强逆转K线测试趋势线极值。
  - 不在1分钟级别反转K线上进场。
  - 单独一根反转K线不足以交易，必须结合前序价格行为。

【K线收盘纪律】
- 不在K线收盘前进场，是逐K分析最重要纪律之一。
- 关键数字：
  - 过早进场一次约多损失8个报价单位。
  - 理论上需8次运行良好的趋势才能对冲一次过早进场损失，实际几乎做不到。
- 常见风险：
  - 假反转陷阱：收盘前看似强反转，最后几秒崩回极点。
  - 好交易被提前洗出：因收盘前提前移动保护止损而被打掉，随后行情按原方向继续。
- 附加规则：
  - 进场K线收盘前不移动初始止损位。
  - 紧盯进场K线及之后1~2根K线收盘。
  - 收盘强势则更愿意保留仓位，收盘弱势则减仓或收紧止损。
  - 在强趋势中可以稍早考虑趋势中进场，但等待收盘通常仍是更优纪律。

【外包线与内含线】
- 外包线 Outside Bar：
  - 定义：当前K线高于前一根高点且低于前一根低点。
  - 含义：方向不确定，多空都更激进但未分胜负；本质上是K线级别凝滞区。
  - 规则：
    - 不宜直接在外包线突破上追单，尤其5分钟级别、外包线很长时。
    - 若必须参与，考虑固定止损或减小仓位以应对止损过远问题。
    - 不确定时最优决策是等待更多价格行为。
  - 特殊场景：
    - 若出现在趋势线突破/趋势轨道线穿越后的强势反转中，且构成二次进场，可成为极佳进场K线。
    - 强趋势中的外包线更像趋势K线，常引发两次进攻。
    - 外包线后若接内含线，形成“内-外-内”（ioi）结构，可按内含线突破方向处理。
    - 凝滞区中部的外包线通常无意义，除非后续短K线提供潮退机会。
- 内含线 Inside Bar：
  - 定义：当前K线高点低于前一根高点，低点高于前一根低点；高/低点可允许与前一根相等。
  - 规则：长K线极值附近的短K线可触发反转，尤其当短K线本身就是短反转K线时。

【二次进场机会 Second Entry】
- 核心原则：
  - 第二次进场几乎永远比第一次更容易盈利。
  - 多空在第二次尝试失败后，市场常反向运行。
  - 识别二次进场的交易者通常更激进、更有把握。
- 价格规则：
  - 若第二次进场价格优于第一次，要提高警惕，这可能是陷阱。
  - 大多数优质二次进场的价格与第一次相同或更差。
  - 市场通常因提供更多信息而要求你接受更差价格。
  - “好价格常对应失败交易”；过于便宜的二次进场常代表你遗漏了信息。
- 实战规则：
  - 若市场在4根或更多逆向K线后给出第一次进场机会，优先等第二次。
  - 牛市中经历4根强熊线或2~3根长熊线后，不应立刻反向做空，应等第二根回撤K线结束。
  - 好结构中的二次进场若仍失败，应先假设自己读错市场。
  - 除非出现楔形（失败的趋势轨道线突破），否则不要做第三次进场。
  - 标准反转结构都可做信号线，但带强反转K线的二次进场是逆势交易中最安全的结构。

【结构K线 vs 信号K线】
- 结构K线 Setup Bar：
  - 几乎任何K线都可能是结构K线，因为下一根可能启动新走势。
  - 结构K线只是潜在信号线。
  - 仅分析结构K线不足以支撑交易；必须与前序K线共同构成连续结构。
- 信号K线 Signal Bar：
  - 使交易者实际准备进场的结构K线。
  - 只有止损入场单被触发时，结构K线才升级为信号线。
  - 即便成为信号线，也未必值得交易（例如铁丝网中的很多信号应忽略）。
  - 价格行为若尚未表明反转/延续概率更高，则信号线本身无意义。
- 转换逻辑：结构K线 → 止损入场单触发 → 信号线；信号线总是事后确认。

【趋势K线的跟随 Follow-Through】
- 核心原则：趋势K线后必须有跟随；无跟随则可能失败。
- 判断标准：
  - 下一根收盘与趋势K线同向 → 有跟随，趋势可能延续。
  - 下一根反向收盘 → 无跟随，趋势K线可能失败。
  - 下一根虽创新高/低但反向收盘 → 弱跟随，谨慎。
  - 连续2~3根同向跟随 → 强跟随，趋势确认。
- 特殊判断：
  - 过长K线（枯竭K线）可能代表趋势接近结束。
  - 突破中的长趋势K线常在下一根失败，把追随者套住。
  - 凝滞区交易日里，长趋势K线失败尤其常见。
  - 一串强趋势K线通常是健康趋势信号，会伴随更极端走势。

【逐棒决策逻辑】
- 每根K线收盘后：
  - 判断K线类型（趋势K线/十字星）。
  - 判断其与前几根的关系（结构K线/信号线/进场K线）。
  - 判断市场状态（趋势中/凝滞区/反转中）。
  - 更新交易计划。
- 决策矩阵：
  - 信号线出现 → 等待收盘确认 → 在极点外1 tick设置止损入场单。
  - 信号线是趋势K线 → 可顺势、仓位略放大、止损略放宽。
  - 信号线是十字星 → 更谨慎、缩仓、等更多确认。
  - 反转K线出现 → 先确认是否位于趋势末端且具备“趋势线突破 + 极点测试”。
  - 外包线出现 → 等方向确认，不在突破上直接追单。
  - 二次进场出现 → 优先参与。
  - 收盘强势 → 保留更多仓位；收盘弱势 → 收紧止损或减仓。
  - 趋势K线无跟随 → 视为潜在失败信号。
  - 凝滞区中 → 不在高点上方买、低点下方卖，等潮退机会。
  - 枯竭K线出现 → 可能是趋势结束前兆，等待更多确认。

【多时间框架中的逐K分析】
- 原则：
  - HTF单根K线在LTF上往往是多根组合；LTF多根在HTF上可能只是一根的一部分。
  - 高质量逐K分析应在主要交易时间框架完成（通常5分钟或15分钟）。
- 时间框架层级：
  - 大一级：定趋势方向和关键支撑/阻力。
  - 主交易级：逐K主战场。
  - 小一级：补充识别内部结构和更早信号。
- 一致性规则：
  - 多时间框架同向 = 最强信号。
  - HTF方向与信号线方向一致 = 顺势可靠性显著提升。
  - HTF方向与信号线方向相反 = 可能只是回调信号，需更谨慎。
- 观察单一图表原则：
  - 实战中应以一种主要图表为执行依据。
  - 同时盯多图容易降低决策速度并制造冲突。
  - 如3分钟止损被打但5分钟未破，应以既定主图（如5分钟）为准。
  - 不要在交易执行时试图调和多个时间框架的不一致。

【第三层分析流程】
- 1 先看大一级框架，确定大趋势与当前位置（区间/趋势/突破位）。
- 2 逐K扫描最新收盘，分类当前K线并评估实体、影线、与前后K线关系。
- 3 若存在潜在信号线，按“趋势K线 > 反转K线 > 十字星”评估质量，并检查是否与大趋势一致。
- 4 信号线极点外1 tick设止损入场单；若属第一次进场，评估是否应等二次进场。
- 5 等待进场K线收盘确认；根据强弱调整仓位与止损。
- 6 观察下一根是否跟随；有跟随可将止损向盈亏平衡推进，无跟随则考虑减仓/退出。
- 7 处理特殊情况：外包线等确认、反转K线需先有趋势线突破+极点测试、枯竭K线防止趋势结束、连续十字星关注序列方向。
- 8 输出当前市场状态、信号质量、入场条件、风险提示与关键价位供阶段二使用。

【第三层例外与陷阱】
- 过早进场：收盘前的完美信号经常在最后几秒失效；严格等收盘。
- 把凝滞区中的反转K线当信号：若与前几根重叠大，则大多无效。
- 把大十字星当信号：实体短但整根很长，本质是一根K线级别的凝滞区。
- 使用1分钟反转K线逆强趋势：大多失败。
- 二次进场价格更优：通常不是优势而是警报。
- 直接做外包线突破：方向不确定且止损过远。
- 趋势K线无跟随：常是假突破。
- 同时观察多时间框架导致执行混乱：必须选定主图并坚持。

【诊断输出格式】
- 完成三层诊断后，必须输出JSON，供程序进入阶段二。

```json
{
  "cycle_position": "spike/micro_channel/tight_channel/normal_channel/broad_channel/trending_tr/trading_range/extreme_tr/unknown",
  "alternative_cycle_position": "第二可能的周期状态（边界模糊时必填，稳定状态可为null）",
  "direction": "bullish/bearish/neutral",
  "diagnosis_confidence": 75,
  "spike_stage": "active/ending/transitioning（仅当cycle_position=spike或micro_channel时必填）",
  "market_phase": "stable/transitioning",
  "transition_risk": "low/medium/high（仅market_phase=transitioning时必填）",
  "detected_patterns": ["wedge", "reversal_attempt"],
  "key_signals": ["信号1", "信号2"],
  "htf_context": "更高时间框架背景描述",
  "entry_setup": "是/否/等待",
  "strategy_files_needed": ["策略文件名1", "策略文件名2"],
  "risk_warning": "需要警惕的风险点"
}
```

【输出字段硬规则】
- diagnosis_confidence：必填整数0~100；表示对 cycle_position 的把握度，不是入场强度。
- 当 diagnosis_confidence < 40 且 key_signals 为空时，阶段二应倾向不下单。
- strategy_files_needed 仅供参考，不得覆盖程序静态路由。

【spike_stage 路由规则】
- active：阶段二加载「极速上涨/下跌策略」+「极速上涨/下跌识别」。
- ending：阶段二加载「极速上涨/下跌策略」+「极速上涨/下跌识别」+ 预加载通道策略文件。
- transitioning：按通道态处理，加载通道策略+通道识别；bullish 加载上涨通道策略/识别，bearish 加载下跌通道策略/识别。

【alternative_cycle_position 路由规则】
- 不为 null 时，阶段二同时加载主状态与备选状态对应文件。
- 例如：normal_channel + trending_tr → 同时加载通道策略与震荡区间策略。
- 实际分析仍以主状态策略为主、备选策略为辅。

【market_phase 路由规则】
- transitioning + high：阶段二强制将仓位缩至正常50%，收紧止损，降低入场频率。
- transitioning + medium：仓位缩至正常75%，选择性入场。
- stable：正常执行。

【detected_patterns 字段】
- 特殊形态不能只写进 key_signals，必须写入 detected_patterns；程序路由主要识别该数组（及 entry_setup_type 补全）。
- bar_analysis.entry_setup_type=wedge → detected_patterns 必含 wedge。
- entry_setup_type=breakout_pullback → detected_patterns 必含 breakout_pullback。
- 多个形态可并存，程序会加载全部对应文件。
- 无特殊形态时填 []。
- 摘要映射：
  - wedge：楔形/三推/趋势线收敛 → 文件14。
  - reversal_attempt / mtr / final_flag：反转尝试/主要趋势反转/末端旗形 → 文件15。
  - h1 / h2 / l1 / l2：计数入场结构 → 文件19。
  - breakout_test / breakout_pullback：突破测试/突破回踩 → 文件18。
  - breakout_failure / failed_breakout：突破失败/假突破 → 文件18、文件22。
  - always_in / ail / ais / 20gb / gap_bar：Always In、20GB、缺口棒 → 文件20。
  - barbwire / wire / overlap / middle_range：铁丝网、重叠、区间中部 → 文件21。
  - failed_signal / magnet / trapped_traders：信号失败、磁力位、被套 → 文件22。

【unknown 兜底规则】
- 当K线数量 < 20 或数据质量不足时，输出 cycle_position="unknown"。
- 程序对 unknown 不加载任何策略文件，阶段二直接输出 action="wait"。
- 此时 diagnosis_confidence 不得超过30。
